# 事例

会則・規則の解釈や運用上の判断に関する記録を保存するディレクトリ

## 目的

- 会則の曖昧な部分に対する解釈の統一
- 過去の施行・判断の参照

## ファイル命名規則

```
YYYY-MM-DD_件名.md
```

## 記録フォーマット

```markdown
# [件名]

## 関連条項

（例: 電子計算機研究会会則 第6条第3項）

## 事案の概要

（何が問題となったかを簡潔に記述）
**このリポジトリは公開されるため，個人情報を記入しない!!!!!!**
個人情報がどうしても必要な場合は部内の内部ドキュメントに追加してください

## 判断・解釈

（どのように解釈・判断されたかを記述）
**このリポジトリは公開されるため，個人情報を記入しない!!!!!!**


## 根拠

（判断の根拠を記述）

## 備考

（その他の情報）
```
